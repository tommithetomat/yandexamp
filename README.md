# YandexAmp ⚡

Десктопный плеер в стиле классического **Winamp 2.x** со стримингом из **Яндекс Музыки**.

## Возможности

**Интерфейс**

- 🎨 Классический вид Winamp 2.x: LCD-дисплей, бегущая строка, обложка трека, плейлист
- 🌈 5 скинов: Classic Green, Blue Steel, Amber, Yandex, Blood — кнопка `CLR` в титлбаре
- ✨ 5 визуализаторов (клик по дисплею): осциллограф, спектр, LED-матрица, фосфорный осциллограф и неоновые полосы
- 🎛 10-полосный эквалайзер с пресетами (Rock, Pop, Classic, Bass)
- 📌 Кнопка `PIN` — окно поверх всех остальных
- 📐 Окно подстраивается по высоте: свернул плейлист или эквалайзер — плеер стал компактнее

**Музыка**

- 📚 Свои плейлисты + подборки Яндекса: Плейлист дня, Премьера, Дежавю
- ❤️ «Мне нравится» — вся библиотека лайков целиком
- ⚡ «Моя волна» — бесконечный поток с настройкой по характеру, разнообразию и языку: треки подгружаются на лету, ротор учитывает пропуски и лайки
- 👍 Лайки и дизлайки прямо из плеера, ♥ у любимых треков в плейлисте
- 🔎 Поиск по всей Яндекс Музыке; если трек уже в плейлисте — плеер переходит к нему, а не дублирует
- 🔀 Shuffle с честной историей: «назад» возвращает на реально игравший трек

**Удобство**

- 🔐 Вход через браузер (OAuth), по логину/паролю или токеном — один раз и надолго
- 💾 После перезапуска на месте: сессия, плейлист, текущий трек, громкость, скин и визуализатор

> Для прослушивания нужна активная подписка Яндекс Плюс.

## Установка (для пользователей)

### Windows

1. Скачайте `YandexAmp-*-Windows-Setup.exe` из [Releases](../../releases)
2. Запустите и подтвердите запрос прав администратора. По умолчанию плеер ставится в `C:\Program Files\YandexAmp`, папку можно выбрать свою
3. Ярлыки на рабочем столе и в меню «Пуск» создадутся автоматически
4. SmartScreen может предупредить о неизвестном издателе — «Подробнее → Выполнить в любом случае»

### macOS

1. Скачайте `YandexAmp-*-macOS-arm64.dmg` (Apple Silicon) или `YandexAmp-*-macOS-x64.dmg` (Intel) из [Releases](../../releases)
2. Откройте DMG и перетащите YandexAmp в Applications
3. Приложение не подписано сертификатом Apple, поэтому при первом запуске: **правый клик → Открыть**. Если macOS пишет «приложение повреждено», выполните в терминале:
   ```bash
   xattr -cr /Applications/YandexAmp.app
   ```

## Запуск из исходников (для разработчиков)

Работает на Windows и macOS:

```bash
git clone <repo-url>
cd yandexamp
npm install
npm start
```

## Сборка установщиков

```bash
npm run dist       # Windows: dist/YandexAmp-<версия>-Windows-Setup.exe
npm run dist:mac   # macOS (только на маке): dist/YandexAmp-<версия>-macOS-<arch>.dmg
```

DMG собирается только на macOS — но для этого есть CI: при пуше тега `v*` GitHub Actions собирает установщики под обе платформы и публикует релиз:

```bash
npm version patch --no-git-tag-version   # поднять версию в package.json
git commit -am "v1.1.4" && git push
git tag v1.1.4 && git push origin v1.1.4
```

## Как это устроено

- **Electron** — безрамочное окно, стилизованное под Winamp; рантайм-зависимостей нет, только встроенные модули Node
- **Web Audio API** — цепочка BiquadFilter'ов для эквалайзера + AnalyserNode для визуализаторов
- **Неофициальное API Яндекс Музыки** (`api.music.yandex.net`) — поиск, плейлисты, лайки, ротор «Моей волны», прямые ссылки на потоки

## Данные и безопасность

- Плеер общается **только с серверами Яндекса** по HTTPS: `oauth.yandex.ru`, `api.music.yandex.net` и CDN с музыкой и обложками. Никакой аналитики и сторонних сервисов
- **Токен авторизации** сохраняется локально в профиле приложения, зашифрованный системным хранилищем ОС (`safeStorage`: DPAPI на Windows, Keychain на macOS) — расшифровать его может только ваш пользователь на этой машине. Наружу он уходит исключительно в запросах к самому Яндексу
- Плейлист, громкость и настройки интерфейса лежат в локальном хранилище приложения. Пароль нигде не сохраняется
- `client_id`, встречающиеся в коде, — публичные идентификаторы официального мобильного приложения Яндекс Музыки, а не чьи-то личные ключи

## Дисклеймер

Проект не аффилирован с Яндексом и использует неофициальное API, поэтому: работоспособность может сломаться при изменениях на стороне сервиса, а само по себе использование такого API расходится с условиями сервиса — теоретические претензии к аккаунту исключать нельзя. Только для личного использования со своей подпиской.
